// 02 · notable findings

Things I've
broken.

Selected reports — redacted where programs request confidentiality. Full details available via HackerOne profile.

P1 · Critical Aug 2024 Shopify

SSRF via webhook URL validation bypass → internal metadata exposure

Webhook endpoint performed URL validation client-side only. By sending a redirect chain through a controlled server, I was able to make Shopify's backend hit 169.254.169.254 (AWS IMDS) and return temporary IAM credentials in the webhook response body. Reported, triaged as critical within 6 hours. Fixed in 3 days.

SSRFAWS IMDSWAF bypassredirect chain
bounty $4,500
P2 · High Mar 2024 GitLab

IDOR in project export API allowing cross-namespace repository dump

The /api/v4/projects/:id/export/download endpoint validated group membership but not project-level visibility overrides. An attacker with any valid GitLab account could enumerate private project IDs and download full repo archives including CI/CD secrets stored in .gitlab-ci.yml. Affects self-hosted and gitlab.com.

IDORbroken authCI/CD secrets
bounty $2,800
P2 · High Nov 2023 Meta (Facebook)

Auth token reuse across Facebook Business Manager sub-accounts

Long-lived access tokens issued to Business Manager apps were not scoped per ad-account. A token from account A could be replayed against account B's ad management API if the underlying user had ever had permissions on both. Exposed campaign data, spend history, and audience targeting of competing brands.

broken access controltoken misuseauth logic
bounty $3,000
P3 · Medium Jul 2023 Google VRP

Subdomain takeover on *.google-analytics.com via orphaned Cloud Storage bucket

Found an unclaimed GCS bucket being referenced by a CNAME on a google-analytics.com subdomain during passive recon. Claimed the bucket and demonstrated full content control. Not weaponisable for credential theft due to SameSite policies but still a brand/trust issue. HoF + swag.

subdomain takeoverGCSrecon
reward HoF + swag
P1 · Critical Feb 2023 [Confidential]

Pre-auth RCE in internal build system via unsafe deserialization

Program name redacted per disclosure agreement. Build orchestration service exposed a debug endpoint accepting serialized Java objects. Sent a crafted ysoserial CommonsCollections payload, got RCE on the build host — which had network access to prod deploy keys. Full write-up pending NDA expiry in Q3 2025.

// 04 · timeline

How I got
here.

2021

First valid bug — IDOR on a food-delivery startup

Found it in my college network lab by accident. Their mobile API used sequential integer IDs for order lookups with zero auth check. Could see every user's name, address, and past orders by incrementing /api/orders/<id>. Got ₹5,000 and a thank-you email. Created my HackerOne account the same night.

first blood
2022

Passed OSCP — 73.5 hours, 4 machines + AD set

Took three months of prep, failed the AD set on my first attempt, came back and got it. The exam environment is genuinely stressful in a way that CTFs aren't. Taught me to document as I go rather than scrambling at the end.

certification
2022

Joined Bugcrowd — first private program invite

Got invited to a fintech private program after a few public reports. Private programs are a different game — smaller scope, more focus, and companies that actually pay fast. Found an auth bypass in their password reset flow (race condition + predictable token) for $1,200.

milestone
2023

Google VRP Hall of Fame + Meta recognition

Two HoF entries in the same quarter. The Google one was a subdomain takeover (boring but valid). The Meta one was a token-reuse issue that I almost didn't report because I wasn't sure if it was in scope. It was. Always report, let the program decide.

hall of fame
2024

Best year — $22,000+ in bounties, P1 at Shopify

The Shopify SSRF was the highlight. Also picked up a pre-auth RCE on a confidential program that I can't name yet — that one's still my best technical find. Crossed Top 500 on HackerOne. Started mentoring two newer hunters informally.

best year
2025

Expanding into cloud-native & AI/LLM attack surface

A lot of programs are deploying LLM features fast and thinking about security second. Prompt injection, insecure RAG pipelines, model exfiltration via tool calls — it's a genuinely interesting new attack surface. Wrote a nuclei template for exposed LLM debug endpoints that's been forked a few times.
